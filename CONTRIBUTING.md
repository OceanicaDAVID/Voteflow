# Contributing to VoteFlow

Thank you for your interest in contributing to VoteFlow! We welcome contributions from the community to help make this decentralized voting platform better.

## Getting Started

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally:
    ```bash
    git clone https://github.com/your-username/voteflow.git
    cd voteflow
    ```
3.  **Install dependencies**:
    ```bash
    npm install
    ```
4.  **Set up environment variables**:
    Copy `.env.example` to `.env` and fill in the required keys (Database, NextAuth, OAuth).
5.  **Run the development server**:
    ```bash
    npm run dev
    ```

## How to Contribute

### Reporting Bugs
If you find a bug, please open an issue on GitHub with:
- A clear title and description.
- Steps to reproduce the issue.
- Expected vs. actual behavior.

### Suggesting Enhancements
We love new ideas! Open an issue to discuss your feature request before starting work.

### Pull Requests
1.  Create a new branch for your feature or fix: `git checkout -b feature/amazing-feature`.
2.  Commit your changes with clear messages.
3.  Push to your fork: `git push origin feature/amazing-feature`.
4.  Open a Pull Request (PR) to the `main` branch of the original repository.

## Code Style
- We use **TypeScript** and **Tailwind CSS**.
- Please ensure your code passes linting: `npm run lint`.
- Follow the existing project structure.
